Confirm completed work actually works. Never trust claims without evidence.

## Rules

- Assume incomplete/wrong until proven. Check happy path AND edge cases (nulls, empty, boundaries, failures).
- Run tests (`flutter test`, project scripts, or manual steps). No "it's done" without checks.

## Steps

1. Identify what was implemented and its acceptance criteria.
2. Run relevant tests. If missing, note gaps and run targeted checks.
3. Flag unverified edge cases.

## Output

Concise report: checks ran, pass/fail, gaps found, follow-ups needed.
