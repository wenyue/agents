# Worktree 环境准备

编写一份完整、由目标项目负责的 Skill，用于准备已经创建的关联 Git worktree，并在仓库可以开始
实现工作时结束。本生成契约不执行环境准备，也不负责 worktree 生命周期。

## 证据

检查目标仓库的 `Project Tools` Rule、项目清单、锁文件、工具链版本配置、准备和生成入口、
CI 工作流、生成源文件策略、必需资产、环境变量、凭据、本地服务和就绪检查。确定：

- 如何找到仓库根目录、Git 公共目录，并在作出任何修改前确认当前目录是已经创建的关联 worktree；
- 每个新 worktree 都需要的最少准备，以及只在特定任务下运行的可选分支；
- 安装锁定依赖、初始化必需输入、生成必需产物或启动必需服务时，可以复用的最小仓库自有命令；
- 项目支持哪些宿主平台、已有哪种脚本运行时、是否能安全重跑、失败时如何处理，以及怎样观察到环境就绪；
- 哪些职责属于 worktree 创建、机器配置、基线验证、业务实现、Agent 同步、集成或清理，而不属于环境准备。

不要臆造目标证据无法证明的跨平台承诺、命令、前置条件或就绪检查。

## 编写流程

1. 根据上述证据，明确生成 Skill 的触发条件、前置条件、就绪状态、停止条件和不负责的生命周期工作。
2. 优先复用范围最小的仓库自有准备入口。只有目标项目缺少可靠入口，无法稳定执行重复、确定的编排时，
   才添加 Skill 自有脚本。
3. 如果确实需要自有脚本，可以选用目标项目已有的语言和运行时，也可以同时生成
   `scripts/setup.ps1` 与 `scripts/setup.sh`。成对入口必须得到同一项受支持的结果，但可以保留经过核实的
   宿主差异。
4. 将确定性的命令序列放进脚本；`SKILL.md` 负责调用方式、前置条件、可选分支、就绪要求、失败恢复和
   结果报告。
5. 不依赖旧版目标 Skill 或 diff，从头阅读完整生成目录。反复调整，直到每条指令和每项资源都只有一个
   明确责任方，也只有一种执行含义。

## 生成 Skill 契约

- 发现元数据只应在准备目标仓库中已经创建的关联 worktree 时触发。正文必须写明环境结果，以及准备
  到哪里结束。
- 在安装依赖、生成文件、变更服务或作出其他准备修改之前，识别并拒绝主检出目录。路径必须从发现的
  仓库根目录或 Skill 根目录解析，不能依赖调用方当前目录。
- 只执行有证据支持、遵守锁文件且由仓库负责的准备工作。昂贵、可选、平台特有或任务特有的分支不得
  进入默认路径，除非每个新 worktree 都必须执行。
- 自有命令序列必须在失败时停止，并能在部分完成后安全重跑。不能悄悄换用未经核实的命令或降级结果。
- 应通过真实项目配置、必需产物和工具或服务行为验证就绪。如果能够检查实际功能，单纯版本探测并不足够。
- 必须包含 `## Failure Recovery`。报告失败步骤、准确命令或条件、可用时的退出状态、相关输出和最小
  修复动作。修改候选脚本后，必须先审查再重试。
- 报告关联 worktree 根目录、完成的准备、选中的可选分支、已经核实的就绪状态，以及有意交给其他
  责任方处理的任务相关准备。
- 环境就绪后立即结束。不得负责 worktree 选择或创建、机器配置、基线或完成变更验证、业务实现、
  Git 历史、集成、清理和 Agent 同步。后续完成变更验证交给 `change-set-verification`。

## 审查关口

实际执行前，从头审查完整生成目录。逐项对照目标证据，确认所有命令、前置条件、修改、平台承诺、
可选分支、就绪检查和职责边界。检查 Skill 自有脚本的确定性顺序、路径解析、失败传递、重跑安全性，
以及与 `SKILL.md` 是否一致。存在过时或无依据的指令，就不能通过审查。

## 验收关口

审查通过后，在目标仓库中一个有代表性、已经创建的关联 worktree 里实际执行完整生成 Skill。
必须调用真实候选版本，证明主检出目录防护会先于任何修改执行，完成默认准备路径，并验证契约声明的
就绪状态。只有目标契约把可选分支列为本次验收能力时，才执行该分支。

项目匹配脚本使用项目已有运行时验证。采用成对入口时，只运行当前宿主对应的入口，不能声称另一个
宿主已经执行。记录准确命令、观察到的修改、就绪证据、停止路径证据和所有未运行项。出现意外修改、
无依据的前置条件或无法证实的就绪状态，都不能通过验收。

## 交接

只有两个关口都通过后，才能把完整、通过验收的目录、仓库证据、审查结论、验收证据，以及未解决或
未运行的平台和可选分支交给 `setup-project-agents`。任一关口失败时都应停止并报告阻碍，不能把候选
版本当作已经验收的产物交接。
