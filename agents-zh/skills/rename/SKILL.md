# 重命名符号

完成一次完整重命名：确定符号身份、更新每个引用、符合项目命名约定并进行验证。

## 工作流

1. **识别**目标符号的名称、种类和作用域。如果作用域有歧义，询问用户，或仅将重命名限制在意图明显的目标上。
2. 使用全词匹配进行全项目**引用查找**。除非用户明确要求，否则忽略生成文件。
3. 按顺序**重命名**：先改声明，再改所有引用。同步更新映射该符号的注释和字符串字面量。除非用户另有说明，否则不要更改 JSON 键、外部 API 字段名和配置键。
4. **约定**：遵循目标项目的类型、成员和文件命名规则。如果周边代码已经偏离默认规则，则匹配项目现有风格。
5. **验证**：运行分析器和相关测试，并修复所有新错误。

## 规则

- 不同作用域中的同名符号应视为独立重命名。除非用户明确要求全项目重命名，否则不要跨作用域应用。
- 对公共 API 中的符号，如果调用方会因此中断，应添加弃用别名；没有约定的迁移路径时不要删除旧名称。
- 对生成文件：重命名源并重新生成；绝不手改生成输出。

## 输出

简短总结：重命名了什么、修改了多少调用位置，以及验证结果。
