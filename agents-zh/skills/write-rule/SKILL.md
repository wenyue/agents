# 编写规则

创建一份无需隐藏上下文即可由其他 Agent 执行、具有充分证据且前后一致的规则。每次调用都重新构建
完整的责任政策，不保留以往编辑形成的结构。

## 分类

编写前根据分发和所有权证据选择一种类型。

| 类型 | 负责内容 | 具体程度 |
| --- | --- | --- |
| 项目本地规则 | 一个仓库的最终政策 | 已验证的仓库相对路径、命令、API、模块、lint、服务和例外 |
| 共享规则 | 直接应用于多个仓库的稳定政策 | 跨仓库不变量、语义化项目目标和协议拥有的路径 |
| 共享规则生成契约 | 编写完整的目标项目自有规则 | 目标证据类别和生成规则的契约，而非最终项目政策 |

目录、清单、安装范围、同步所有权、Wrapper 和请求的输出比泛化措辞更能证明分类。删除本地细节
不会让规则自动成为共享规则；应围绕真正稳定的政策重新设计，否则保留为项目本地规则。

## 证据

只收集会改变规则内容、所有权或验证方式的证据：

- 用户的目标、约束、示例、技术选择、阈值和兼容性要求；
- 完整的责任规则族、适用的更宽泛和更具体规则，以及当前优先级；
- 清单、目录、Wrapper、镜像、验证器、测试和真实用法；
- 对项目本地规则，检查当前代码、配置、schema、命令、服务、lint、生成内容所有权和模块边界；
- 对共享文件，找出在所支持仓库中始终成立的最小政策或证据类别。

把现有规则当作证据和防遗漏检查，而不是首选提纲。区分稳定政策与变更历史、个人偏好、可复用流程，
以及由其他位置负责的运行时事实。

## 编写

1. 明确规则完整负责的政策、排除的职责，以及支持其类型、强度和适用范围的证据。
2. 从当前意图综合完整候选文件。保留仍有依据的方向、阈值、技术选择和例外；删除过时、重复、矛盾或
   位置错误的内容。
3. 将每项要求放到自然所有者中。只有请求文件确实负责该政策时才更新它；否则更新真正的所有者，
   或在迁移会实质改变请求时询问用户。
4. 在适用范围或分发需要时，同步更新所属的 Wrapper、索引、清单、镜像和契约测试。
5. 脱离 diff 阅读完整候选文件，持续修改，直到每段内容都提供独立、现行、可执行的信息。

优先描述目标状态、可观察条件和所有权，不叙述不必要的流程。多种做法都正确时给执行 Agent 留出自由；
顺序本身属于政策或几乎没有有效优化空间时再规定步骤。示例只用于简洁消除歧义。

受版本控制的规则只使用仓库相对路径或协议拥有的稳定路径，不使用机器特有的绝对路径。每项要求只保留
一个事实源；Wrapper 只包含平台必需元数据和一个源引用。

引用其他 Rule 或 Skill 时，使用目标系统声明或识别的规范名称，绝不使用其文件系统路径。只有当前契约
所拥有且位置属于契约的文件或资源才使用路径。

## 规则契约

目标使用本仓库格式时，以以下内容开头：

```markdown
# Rule Title

Strength: `Mandatory|Default|Advisory`

Scope: One sentence naming the rule's owned responsibility.
```

使用目标系统支持的标题、强度、范围、编号或适用声明，以及优先级词汇。只有职责需要时才增加章节，
不要为了满足通用模板而增加章节。

| 契约结构 | 必需内容 | 额外要求 |
| --- | --- | --- |
| 项目本地直接规则 | H1；Strength；Scope；按职责组织的政策；必要时增加 Boundaries 或 Exceptions | 陈述已验证的目标政策及其关系 |
| 共享直接规则 | H1；Strength；Scope；稳定政策；必要时增加 Boundaries 或 Precedence | 将目标项目特有政策交给项目本地规则 |
| 共享规则生成契约 | Generation Contract；Evidence；Content；Boundaries | 生成完整的目标项目自有候选文件，不混合编写说明和最终目标政策 |

宽泛规则不得重复或静默覆盖更具体的规则。

## 验证

- 阅读最终规则而不只看变更行，验证分类、所有权、强度、范围、适用条件、优先级、例外和完整政策。
- 对项目本地规则，根据当前仓库的强制入口核对具体声明和跨规则关系。
- 对共享规则，在包含项目本地优先级和覆盖项的代表性环境中测试。
- 对生成契约，至少模拟一份完整目标规则；声称广泛可移植时使用差异足够大的多个目标。
- 按结构比较语言镜像，保留路径、命令、标识符、代码块、分类、强度和行为。
- 对所有所属发现入口运行当前验证器、契约测试和 diff 完整性检查。

证据仍未解决、声明缺乏支持、所属入口过时，或必需检查失败或未报告时，不得声称成功。

## 结果

报告文件类型、自然所有者、最终契约结构、保留的决定、删除或迁移的内容、更新的发现入口和镜像，
以及准确的验证结果。
