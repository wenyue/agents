# 响应格式

强度：`Default`

适用范围：助手响应语言、标签协议、格式以及 review/实现报告。

## 语言

- 每次用户请求后的第一条助手消息用英文重述用户请求。
- 最终回复第一行用英文重述用户请求。
- 中间进度更新不重复英文重述，应使用简体中文并保持简洁。
- 除非用户明确要求其他语言，其余面向用户的文本均使用简体中文。

## 标签协议

非简单回复使用以下 `##` 标题；空标题省略，极短回复使用普通文本。

| 标签 | 用途 |
| --- | --- |
| `🎯` | 用英文重述用户目标。 |
| `⚠️` | 风险、约束、前置条件或重要假设。 |
| `✅` | 已完成结果、修改文件和验证。 |
| `❌` | 失败或 blocker，以及继续所需条件。 |
| `🤖` | 用户交互：一个明确问题或少量选项。 |

推荐顺序：

```text
🎯 → ⚠️ → ✅ or ❌ → 🤖
```

## 约束

- 使用 `🎯` 时必须放在最前面，内容必须是英文。
- 仅在存在实质风险或假设时使用 `⚠️`，最多三项。
- `✅` 和 `❌` 不得同时出现。
- `🤖` 必须是最后一节；提出问题后停止回复。
- 默认不要添加分析或规划章节。必要的权衡与决定放在 `⚠️` 或普通正文中。

## 通用格式

- 当有助于浏览时使用 Markdown 的粗体、行内代码、引用、表格和代码块。
- 不要使用“好的”“明白了”“当然”等低信息开场。
- 叙述内容优先使用短段落而不是长列表。
- 路径、命令、symbol、配置 key 和字面量使用行内代码。
- 实现工作应说明修改文件和验证；未运行或被阻塞的验证必须明确说明。
- Review 应先列发现，按严重程度排序，并尽可能提供文件和行号。
- 计划或设计说明使用中文，并明确权衡。

## 标签指南

### 🎯 目标

用于每次用户请求后的第一条回复，或最终回复中必须包含的英文目标重述。中间进度更新不要使用。

```markdown
## 🎯
#### Update the menu owner scope behavior.
```

### ⚠️ 警告

仅用于有实质意义的风险、约束、前置条件或假设。

```markdown
## ⚠️
> **验证限制** — Widget 行为仍需要一个聚焦的回归测试。
```

### ✅ 完成

工作完成时使用。相关时包含修改文件和验证。

```markdown
## ✅
已更新 `menu_owner_scope.dart`，让 owner 变更时正确刷新菜单状态。

验证：已运行 `flutter test test/common/widgets/menu_owner_scope_test.dart`。
```

### ❌ 阻塞

任务无法完成时使用。

```markdown
## ❌
> **缺少复现条件** — 当前信息不足以定位菜单状态不同步的触发路径。

需要提供具体操作步骤或失败测试。
```

### 🤖 交互

当歧义、权衡、范围扩展、约定偏离或缺失信息需要用户输入时使用。

只提出一个问题，或给出少量选项和推荐。出现 `🤖` 后立即结束回复。
