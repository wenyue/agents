# 项目规则

强度：`Default`

适用范围：项目 API、领域约定、生成文件归属、lint 解释、持久化和生命周期行为的生成契约。

## 生成契约

根据稳定的行为证据编写目标规则。记录实现和审查 Agent 必须维持的约束；省略仅仅属于常见做法
或个人偏好的约定。

## 证据

- 公共 API、路由、schema、事件定义、序列化代码和兼容性测试。
- 框架配置、已有调用点、自定义 lint 和仓库专用 analyzer。
- 生成文件头、源 schema、生成器配置和归属文档。
- 领域模型、持久化和迁移代码、生命周期所有者及并发边界。
- 反复出现的命名、术语、本地化和面向用户的文案约定。

## 内容

- 记录公共 API、路由、事件、payload 和兼容性契约。
- 记录项目专用的框架用法，以及对 formatter、analyzer 和 lint 输出的解释。
- 记录生成文件和外部 schema 的语义所有者、重新生成要求，以及禁止手工编辑的文件。
- 记录领域术语、命名约束、前缀、标识符和面向用户的文案约定。
- 记录持久化、迁移、状态归属、生命周期、取消和并发规则。
- 只有当项目当前证据能够证明真实覆盖关系时，才记录对基础规则的例外。

## 边界

- 工具调用、生成命令、运行时和验证能力应放在 `Project Tools` 中。
- 目录归属、模块布局和依赖方向应放在 `Project Structure` 中。
- 不要包含基础规则已经覆盖的通用语言风格，也不要加入缺乏依据的架构建议。
- 不要在 `Project Tools` 与本规则中重复生成文件事实。本规则负责语义归属和编辑边界；
  工具规则负责生成器调用。
