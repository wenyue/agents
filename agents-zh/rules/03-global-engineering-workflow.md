# 工程工作流

强度：`Mandatory`

适用范围：工程任务的理解、准备、修改和验证。

## 理解问题

- 明确期望结果、约束和验收条件，并以解决根本问题为目标。
- 用户的表达存在歧义时，应先向用户确认再继续处理。
- 对于咨询或信息类问题，回答前先搜索网上资料，并以当前来源作为依据。
- 编辑前，先阅读适用规则、相关文件和附近代码。
- 对于非简单改动，找出根因、不变量、依赖关系、风险和影响范围。

## 修改

- 选择能够解决根因、符合周边设计的最小完整改动。
- 将改动限制在请求范围内。扩大范围前，先提出更完整的方案及其重要取舍。
- 沿用已有模式和归属边界。仅在当前需求确有需要时引入新结构。
- 必须采用变通方案时，限制其影响范围并明确其局限。
- 删除因本次改动而过时的代码。

## 验证

- 完成工作前，使用与任务及其风险相称的检查验证修改后的行为。
- 覆盖请求结果、适用时的原始问题，以及可能的副作用。
- 验证发现失败时，回到诊断阶段，依据新证据修正判断和方案。
- 针对同一问题连续尝试两次仍失败后，停止重复当前做法，并找出阻塞点、证据和下一步有效措施。
