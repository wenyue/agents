# C++ 指引

强度：`Default`

适用范围：工具配置之外的 C++ 归属、接口、失败契约和并发约束。

## 接口与归属

- 保持公共接口精简，并维持既有 ABI、FFI 和平台边界。
- 行为应留在其所有者中；只有一组相关输入形成稳定概念时，才引入选项类型。
- 优先使用 RAII 和值语义。
- 优先使用智能指针，不要使用负责所有权的裸指针。

## 失败与并发

- 将预期失败纳入返回契约；异常只用于周围边界确实视为异常的失败。
- 通过一个明确的所有者和范围最窄的合适原语来同步共享可变状态。
- 跨原生回调维持清理、取消和线程亲和性要求。

## 工具归属

- 机械样式和命名交给仓库的 formatter、编译器和静态分析配置。
- 在原生归属、失败处理或 ABI 行为可能回归的边界测试行为。
